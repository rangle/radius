# Getting Started

Explore UI components Click on any of the component in the sidebar to see how they look, and how to configure them. Each component pages contains documentation of its properties, as well as code examples for how to use it.

#### Try them out

Experiment with different properties, by clicking Canvas at the top of each component page.
Change the properties under Knobs in the bottom panel.
Use them Once you've found the right component for your use case, click the Story tab to see the code implementation, or look at examples under Docs.

#### Reporting a Bug
If you spot a bug, inconsistency, or typo, please open a bug issue. Better yet, submit a pull request to address it.

#### Feature Requests
If you have an idea, we would love to hear about it. The best way to suggest a feature is to open a feature issue. 

#### Contributing
Please read our contributing guidelines to find out more and how to get started.

#### Versioning
Design System releases follow semantic versioning and is enforced automatically by conventional commits (see "Commit Message Format").
